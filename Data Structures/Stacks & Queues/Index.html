Stacks & Queues

Linear Data Structures

  • Allow us to traverse - go through data elements sequentially one by one. Only one dat aelment can be directly reached.
  • implemented in similar ways
  • Difference is how items get removed from these data structure.
  • Unlike Array - In Stacks and Queues there is no random access operation.
  • Commands - Push, Peak, Pop all work exclusively with the element at the beginning or the end of the data structure.

STACKS (LIFO)

Stacks a type of data structure - you can think of them as plates. One piece of data and then another piece of data gets stacked on top of that data and keeps going. you can only touch the top plate. you cant really access anything from the bottom.

The way you can get access to the data from a STACK is to grab the first plate, then the second plate, then the third and so on... until youve gone through through all the stack.

LIFO

LAST IN, FIRST OUT

Commands:

  • Push() - Pushing (store) an item on the stack
  • Pop() - removing (access) an item from the stack
  • Peek() - Get the top data element of the stack, without removing it.
  • Isfull() - Checks if the queue is full
  • Isempty() - Checks if the queue is empty
  • Count() - It returns the total number of elements avaible in the stack

QUEUES (FIFO)

Follows a particular order in which the operations are performed. The order is First In First Out (FIFO). A good example of a queue is any queue of consumers for a resource where the consumer that came first is served first. The difference between stacks and queues is in removing. In a stack we remove the item the most recently added; in a queue, we remove the item the least recently added.

FIFO

FIRST IN, FIRST OUT

Commands:

  • Enqueue() - add (store) an item
  • Dequeue() - remove (access) an item
  • Isfull() - Checks if the queue is full
  • Isempty() - Check if the queue is empty
  • Size() - Checks the size of the queue
