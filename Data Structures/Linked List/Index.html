Single or Double LL


SLL

PROS
  • Simple to implement
  • Requires Less Memory - less Operations
  • Faster
CONS
  • Can not be iterated in reverse ir traverse from back to front
  • If we lose the reference to the head node of the list, the list can actually be lost in memory forever.

So Singly List are appropriate to use when you have less memory or memory is really expensive and want to be careful of how much you use. Main goal is for fast insertion, deletion and dont have much searching, especially when you have insertion at the beginning of the list.


DLL

PROS
  • It Can be iterated a or traversed both from the front or from the back
  • Extra operations that will be needed to perform to make sure that when we insert and delete that the previous property is updated as well.
CONS
  • Fairly Complex - requires more memory and storage because of the extra property.
  • If we lose the reference to the head node of the list, the list can actually be lost in memory forever.

So Doubly Linked List are really good when you dont have that much limitation on memory and when you want good operation for searching for elements such as searching backwards instead of just forwards.
